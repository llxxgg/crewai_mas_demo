# 甲方画像（Manager 视角）

## 基本信息

- **角色**：产品负责人（Product Owner）/ Human 甲方
- **沟通渠道**：通过 `main.py` 发起任务；通过 `human_cli.py` 查看和确认消息
- **消息读取路径**：`workspace/shared/mailboxes/human.json`

## Human 的工作习惯

- **异步确认**：Human 不会一直守着终端。Manager 发出通知后，Human 在自己方便的时候打开 `human_cli.py` 查看
- **只处理关键决策**：Human 不需要知道 PM 在做什么。只关心：需求是否正确、SOP 是否合适、最终交付物是否满意
- **反馈简洁**：Human 拒绝时会给出修改意见，Manager 需要基于反馈修订后再通知

## 对 Manager 的期望

| 期望 | 说明 |
|------|------|
| 需求澄清到位 | 澄清问题精准，不问废话，3轮内收敛 |
| SOP 选择有理由 | 通知 Human 时说明选了哪个 SOP 及原因 |
| 通知及时 | 每个重要节点完成后主动通知，不让 Human 主动来问 |
| 验收有结论 | 不要模糊——要么「通过」要么「需返工 + 具体原因」 |
| 不过度打扰 | 团队内部协作不需要 Human 知道；只在决策点通知 |

## 需求输入习惯

- 需求通常以对话形式给出（通过 `main.py` 的 user_request 参数）
- 可能不完整，需要 Manager 主动澄清
- Human 期望被问：目标、边界、约束、风险

## 消息格式偏好

Human 看到的 human.json 消息应简洁清晰：
- 主题：直接说明要确认什么
- 内容：关键路径 + 1-2 句说明，不要长篇大论
