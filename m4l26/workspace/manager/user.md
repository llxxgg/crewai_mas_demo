# 甲方画像（Manager 视角）

## 基本信息

- **角色**：产品负责人（Product Owner）
- **团队**：XiaoPaw 产品团队
- **语言**：中文
- **沟通渠道**：通过 `main.py` 发起任务，通过 `review_result.md` 接收验收结论

## 需求输入习惯

- 需求文档已放在 `/mnt/shared/needs/requirements.md`，**无需额外澄清，直接读取**
- 需求通常包含：项目背景、核心功能、验收标准
- 优先级排列：用户体验 > 功能完整性 > 技术指标

## 对 Manager 的期望

| 期望 | 说明 |
|------|------|
| 工作区结构完整 | Manager 负责用 `init_project` 建好协作基础设施 |
| 任务分配及时 | 需求到了，邮件应该当轮就发出去 |
| 验收有结论 | 不要模糊——要么「通过」要么「需返工 + 具体原因」|
| 路径对得上 | 邮件里的路径引用必须是实际存在的文件路径 |

## 沟通偏好

- **喜欢**：Manager 在验收报告里用表格列出每个检查项，清晰可追溯
- **不喜欢**：验收报告里只有一句「整体不错」——没有具体检查项无法判断质量
- **欣赏**：Manager 能在 `review_result.md` 中指出具体的「哪个用户故事缺少验收标准」，而不只是「文档需要完善」
