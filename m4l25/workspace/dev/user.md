# Dev 工作上下文

> Dev 的直接输入是任务单（feature_requirement.md），由 Manager 分配，不直接对接甲方。
> 本文件记录团队约定和常见输入特征，帮助 Dev 更好地解读任务单。

## 输入特征

- **来源**：Manager 拆解后的任务单，通常已包含功能描述 + 验收标准
- **常见格式**：Markdown，包含「功能描述 / 输入输出格式 / 验收标准 / 技术约束」四项
- **质量参差**：任务单有时会省略技术约束，Dev 按团队默认约定（Python + 类型注解 + 中文注释）处理

## 团队技术约定

| 约定项 | 默认值 |
|--------|--------|
| 编程语言 | Python 3.10+ |
| 类型注解 | 必填，函数签名和返回值都要有 |
| 代码注释 | 中文 |
| 单元测试框架 | pytest |
| 错误处理 | 自定义异常类，不裸 raise Exception |
| 时间格式 | ISO 8601（带时区，+08:00）|

## 常见任务类型

| 类型 | 典型特征 | Dev 注意点 |
|------|---------|-----------|
| NLP 解析模块 | 中文输入，结构化输出 | 优先规则引擎，有性能约束时排除外部 API |
| API 集成 | 调用第三方 API | 必须处理网络错误和速率限制 |
| 数据处理 | 批量转换或清洗 | 关注边界：空输入、超大输入、格式异常 |
| 工具函数 | 复用性强的小模块 | 接口设计要稳定，实现可替换 |
