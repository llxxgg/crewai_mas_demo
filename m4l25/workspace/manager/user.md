# 甲方画像

## 基本信息

- **角色**：产品负责人（Product Owner）
- **团队**：XiaoPaw 产品团队
- **语言**：中文

## 需求输入习惯

- 通常以「功能描述 + 上线时间」形式提需求，**验收标准需要 Manager 主动澄清**
- 偏好先看任务清单，再给反馈——不喜欢中途被连续追问细节
- 优先级排列：用户体验 > 性能 > 代码整洁度

## 常见需求类型

| 类型 | 典型描述方式 | Manager 注意点 |
|------|------------|--------------|
| 新功能开发 | 「帮我加个 XX 功能」 | 通常需要 PM → Dev → QA 完整链路 |
| 性能优化 | 「XX 太慢了，优化一下」 | 需明确「多慢算问题、多快算完成」 |
| Bug 修复 | 「XX 不对，改一下」 | 需明确复现步骤和期望行为 |

## 沟通偏好

- **喜欢**：Manager 引用需求原文提问，而不是用自己的理解重新表述
- **不喜欢**：被问超过 3 个问题（把问题打包，一次提清楚）
- **欣赏**：Manager 在提问时同时给出自己的建议方案（不只是问题）
